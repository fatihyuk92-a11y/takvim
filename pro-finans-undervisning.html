Python & data

Excel går død når du rammer 100.000 rækker.

Det er hvor pandas tager over. Hvis du har lavet finansiel analyse i Excel længe nok, har du oplevet at filer crasher, formler bryder, og du kan ikke versionere noget meningsfuldt. Python med pandas løser det hele — men kræver en ny mental model.

Det minimale Python-finans setup.

Python 3.12+

Brug venv eller uv. Aldrig system Python.

pandas + numpy

Datafundament. pandas til DataFrames, numpy til numeriske operationer.

yfinance + pandas-datareader

Gratis markedsdata. yfinance til daglig data, datareader til økonomiske indikatorer.

matplotlib + seaborn

Grafer. Plotly hvis du vil have interaktive.

Jupyter eller marimo

Notebooks. Marimo er nyere alternativ med reaktiv kørsel.

Første seriøse dataanalyse.

Hent og analysér prisdata
import yfinance as yf import pandas as pd import numpy as np # Hent 10 års daglig data for NVDA, AAPL, MSFT tickers = ['NVDA', 'AAPL', 'MSFT'] data = yf.download(tickers, start='2014-01-01', end='2024-12-31') # Adj Close giver os justerede priser (efter splits/udbytte) prices = data['Adj Close'] # Daglige procentvise afkast returns = prices.pct_change().dropna() # Grundlæggende statistik stats = pd.DataFrame({ 'mean_daily': returns.mean(), 'std_daily': returns.std(), 'sharpe': (returns.mean() / returns.std()) * np.sqrt(252), 'max_dd': (prices / prices.cummax() - 1).min(), 'total_return': (prices.iloc[-1] / prices.iloc[0]) - 1, }) print(stats.round(4))

De fem datafælder i finans.

  • Lookahead bias. Du bruger Q3 EPS i en strategi der handler på den dato Q3 EPS rapporteres. Men i virkeligheden kender du den først dagen efter market close.
  • Survivorship bias. Du backtester på nuværende S&P500. Men 30% af 2014-firmaerne er ikke der i dag. Du tester på vindere.
  • Splits og udbytte. Brug ALTID adjusted prices til tilbageblik. Raw prices ser dramatiske drop ud (faktisk splits).
  • Time zones. US-marked lukker kl 22 dansk tid. Hvilken dato har den lukke-pris?
  • Outliers og fejl-data. En enkelt 1000% spike i data ødelægger dine statistikker. Plot data før du regner.

Datakilder — hvad du får og koster.

yfinance (gratis)

God til læring. Daglige data. Ikke til serious live trading — rate-limited og kan ændres.

Alpha Vantage

Gratis tier OK til hobby. Real-time kræver betalt plan.

Polygon.io

$30+/md. Aktier, options, intraday. God til seriøse backtests.

Databento / Norgate

Professionel data. Survivorship-bias-fri. $100-500/md. Til seriøs kvant.

IBKR Trader Workstation API

Direkte fra din broker hvis du handler hos Interactive Brokers. Real-time og handler.

Datahåndtering — parquet over CSV.

For store datasæt: drop CSV. Parquet er kolonne-orienteret, komprimeret, og 10-100x hurtigere at læse:

Effektiv datalager
# Gem hentet data lokalt for hurtig genindlæsning prices.to_parquet('data/prices_sp500_2014_2024.parquet') # Næste gang prices = pd.read_parquet('data/prices_sp500_2014_2024.parquet') # For store datasæt: brug DuckDB import duckdb con = duckdb.connect('finance.duckdb') con.execute(''' CREATE TABLE prices AS SELECT * FROM 'data/*.parquet' ''') # Nu kan du køre SQL på dine prisdata result = con.execute(''' SELECT ticker, AVG(returns) as avg_ret FROM prices WHERE date >= '2020-01-01' GROUP BY ticker ORDER BY avg_ret DESC LIMIT 20 ''').fetchdf()

Vectorization — regel #1 i pandas.

Loops er død langsomme. Vectorized operationer er 100-1000x hurtigere:

✗ Generisk

Dårligt (loop): for i in range(len(df)): df['sma'][i] = df['close'][i-20:i].mean()

✓ God

Godt (vectorized): df['sma'] = df['close'].rolling(20).mean()

Typiske finansielle beregninger.

Reusable finance helpers
def compute_returns(prices, log=False): """Daglige afkast. Log eller simpel.""" if log: return np.log(prices / prices.shift(1)).dropna() return prices.pct_change().dropna() def rolling_volatility(returns, window=20, annualize=True): """Rullende volatilitet (annualiseret).""" vol = returns.rolling(window).std() if annualize: vol = vol * np.sqrt(252) return vol def max_drawdown(prices): """Maksimum nedstilling.""" cummax = prices.cummax() drawdown = (prices - cummax) / cummax return drawdown.min() def sharpe_ratio(returns, risk_free=0.02): """Annualiseret Sharpe ratio.""" excess = returns - risk_free / 252 return (excess.mean() / returns.std()) * np.sqrt(252) def beta(returns_asset, returns_market): """Beta vs benchmark.""" cov = returns_asset.cov(returns_market) var = returns_market.var() return cov / var

Hvor AI fejler

  • For meget tid på yfinance. Gratis er fint til læring, men ikke noget du satser på i produktion.
  • Loops i pandas. Vectorize alt. Hvis det ikke kan vectorizes, brug numba eller cython.
  • Ignorerer survivorship bias. Du backtester på nuværende index og ser strategier der "altid virker".
  • Gemmer som CSV. Langsomt, stort. Parquet eller DuckDB.
  • Excel dør ved 100k rækker. Skift til pandas tidligere end du tror.
  • Survivorship bias er den vigtigste fælde i backtest-data.
  • Vectorize alt — loops er død langsomme i pandas.
  • Parquet + DuckDB til seriøse datasæt.

Backtest-framework

De fleste backtest-resultater er løgne.

Ikke fordi backtesteren løj, men fordi datakilden og logikken har skjulte bias. Et backtest med 40% årligt afkast lyder fantastisk — indtil du opdager at det inkluderer transaktionsomkostninger på 0%, perfekt liquiditet, og at strategien handler på daglig open med viden om daglig close. Et professionelt backtest håndterer det hele eksplicit.

De seks vigtige aspekter af et seriøst backtest.

Lookahead-fri data

Du må kun bruge data der var tilgængelig på beslutnings-tidspunktet.

Transaction costs

Bid-ask spread, kommission, market impact. Typisk 5-25 bps per handel.

Slippage

Du får ikke prisen du sigtede mod. Modeller realistic udførelses-pris.

Borrow costs

Hvis du shorter, koster det rentebetaling på lånet.

Survivorship-fri univers

Inkludér firmaer der er blevet delistet, gået konkurs, opkøbt.

Realistic position sizing

Du kan ikke handle 50% af daglig volumen uden at flytte prisen.

De tre backtest-frameworks — vurderet.

Backtrader

Klassisk Python framework. God til læring. Documentation kunne være bedre.

Vectorbt / vectorbt-pro

Vectorized, hurtigt. Bedst til at teste mange parametre. Pro-version kræver licens.

Zipline (Quantopian)

Tidligere standard. Vedligeholdes ikke aktivt. Ikke anbefalet til nye projekter.

Custom (numpy + pandas)

For ægte kvant: skriv dit eget. Fuld kontrol, fuld ansvar. Anbefales over tid.

Et minimalt eget backtest-framework.

Backtest-skelet
import pandas as pd import numpy as np from dataclasses import dataclass from typing import Callable @dataclass class BacktestConfig: initial_capital: float = 100_000 commission_bps: float = 5 # 0.05% pr. trade slippage_bps: float = 3 position_sizing: str = 'equal_weight' # eller 'risk_parity', 'kelly' max_position_size: float = 0.10 # max 10% i én aktie class Backtester: def __init__(self, prices: pd.DataFrame, config: BacktestConfig): self.prices = prices self.config = config self.cash = config.initial_capital self.positions = {} # ticker -> shares self.history = [] def run(self, strategy: Callable, signals_lookback: int = 252): """Kør backtest. Strategy returnerer target_weights pr. dato.""" dates = self.prices.index[signals_lookback:] for date in dates: # KRITISK: kun data FØR denne dato historical = self.prices.loc[:date].iloc[:-1] # Strategi-signaler baseret på fortid target_weights = strategy(historical) # Rebalance til target self.rebalance(date, target_weights) # Log portfolio-værdi self.history.append({ 'date': date, 'value': self.portfolio_value(date), 'cash': self.cash, 'positions': dict(self.positions), }) return pd.DataFrame(self.history) def rebalance(self, date, target_weights): """Justér positions til target_weights.""" total_value = self.portfolio_value(date) current_prices = self.prices.loc[date] for ticker, target_pct in target_weights.items(): target_value = total_value * target_pct current_shares = self.positions.get(ticker, 0) target_shares = target_value / current_prices[ticker] delta = target_shares - current_shares if abs(delta) < 1: continue # Økonomisk effekt med costs trade_value = delta * current_prices[ticker] cost = abs(trade_value) * (self.config.commission_bps + self.config.slippage_bps) / 10000 self.cash -= trade_value + cost self.positions[ticker] = target_shares def portfolio_value(self, date): prices = self.prices.loc[date] equity = sum(shares * prices[t] for t, shares in self.positions.items()) return self.cash + equity

Den klassiske overfitting-fælde.

Du tester 500 parametre. Tre virker fantastisk i backtest. Du vælger den bedste. I produktion fejler den, fordi du selectede på støj, ikke signal.

✗ Generisk

Overfittet: "Optimér alle parametre på alle data. Bedste resultat: 45% årligt afkast."

✓ God

Robust: "Train på 2014-2018. Vælg parametre. Test ud-af-sample på 2019-2024. Resultat: 12% afkast — måske ægte signal."

Walk-forward analyse — guldstandarden.

Mest robust målemetode: rullende re-optimering. Strategien skal kunne tilpasse sig over tid uden at være fit til støj:

Walk-forward backtest
def walk_forward(strategy_factory, prices, train_years=2, test_years=1): """Train på N år, test på N+1. Skub frem. Gentag.""" results = [] start = prices.index[0] end = prices.index[-1] current = start while current + pd.DateOffset(years=train_years+test_years) <= end: train_end = current + pd.DateOffset(years=train_years) test_end = train_end + pd.DateOffset(years=test_years) train_data = prices.loc[current:train_end] test_data = prices.loc[train_end:test_end] # Optimér på train best_params = optimize_strategy(strategy_factory, train_data) # Test på unseen data strategy = strategy_factory(best_params) test_result = run_backtest(strategy, test_data) results.append({ 'train': (current, train_end), 'test': (train_end, test_end), 'params': best_params, 'oos_return': test_result['total_return'], 'oos_sharpe': test_result['sharpe'], }) current = train_end # skub frem ét test-vindue return pd.DataFrame(results)

De vigtigste metrikker.

  • CAGR. Compound Annual Growth Rate. Den årlige geometriske afkast.
  • Sharpe ratio. Afkast pr. enhed risiko. >1.0 godt, >2.0 fremragende, >3.0 sandsynligvis overfittet.
  • Max drawdown. Største fald fra peak. Vigtigere end gennemsnit — du må kunne leve med det.
  • Sortino ratio. Som Sharpe men kun straffer negative afvigelser.
  • Calmar ratio. CAGR / max drawdown. Ändrer fokus til "afkast pr. tab".
  • Win rate & profit factor. % vundne handler & (sum gevinster / sum tab).

Hvor AI fejler

  • Optimerer på in-sample data. Sharpe 4.0 i backtest, -0.2 i live trading.
  • Glemmer transaktionsomkostninger. Hyppige handler ser strålende ud uden 5 bps per trade.
  • Ignorerer survivorship bias. Strategi virker fordi du tester på vindere.
  • Ingen walk-forward. Single-period backtest siger næsten intet om fremtidig performance.
  • Antag at backtests overvurderer afkast med faktor 2-5.
  • Walk-forward er det eneste pålidelige test.
  • Inkluder altid transaktionsomkostninger og slippage.
  • Sharpe > 3 i backtest er næsten altid overfitting.

Alpha-strategier

Det meste "alpha" er beta i forklædning.

Hvis din strategi har 0.7 korrelation med S&P500, klarér du dig godt når markedet gør — men det er ikke alpha. Det er bare leverage. Ægte alpha er afkast der ikke kan forklares af kendte faktorer (markedet, størrelse, value, momentum). Det er sjældent, svært at finde, og endnu sværere at beholde.

De fire strategi-kategorier.

Momentum

Køb det der stiger. Klassisk faktor med veldokumenteret premium.

Mean reversion

Køb det der er faldet, sælg det der er steget. Virker bedst på kort tid.

Statistical arbitrage

Find par/grupper der historisk bevæger sig sammen, handl divergens.

Risk premia

Sælg "forsikring" via volatilitet, carry, default risk. Lange perioder gode, så katastrofe.

Momentum-eksempel — simpelt men effektivt.

12-1 momentum strategi
def momentum_signal(prices, lookback=252, skip=21): """Klassisk 12-1 momentum: 12 mdr afkast minus seneste måned. Beviset for at undvige seneste måned undgår mean-reversion. """ # Pris for 252 dage siden vs 21 dage siden returns_12m = prices.shift(skip) / prices.shift(lookback) - 1 return returns_12m def momentum_strategy(prices, top_n=20, hold_months=1): """Hver måned: køb top 20 momentum-aktier, equal weight.""" monthly_prices = prices.resample('M').last() signals = momentum_signal(monthly_prices) weights = pd.DataFrame(0.0, index=signals.index, columns=signals.columns) for date in signals.index[lookback:]: # Vælg top N top_stocks = signals.loc[date].nlargest(top_n).index weights.loc[date, top_stocks] = 1.0 / top_n return weights

Mean reversion på par.

Pairs trading (cointegration)
from statsmodels.tsa.stattools import coint def find_cointegrated_pairs(prices, significance=0.05): """Find aktier der er kointegrerede (long-run sammenhænge).""" pairs = [] tickers = prices.columns for i in range(len(tickers)): for j in range(i+1, len(tickers)): t1, t2 = tickers[i], tickers[j] score, pvalue, _ = coint(prices[t1], prices[t2]) if pvalue < significance: pairs.append((t1, t2, pvalue)) return sorted(pairs, key=lambda x: x[2]) def pair_signal(price_a, price_b, lookback=30, entry_z=2.0): """Z-score af spread.""" spread = price_a - hedge_ratio(price_a, price_b) * price_b rolling_mean = spread.rolling(lookback).mean() rolling_std = spread.rolling(lookback).std() z = (spread - rolling_mean) / rolling_std # Long A / short B når z < -2, modsat når z > 2 signal = pd.Series(0, index=z.index) signal[z < -entry_z] = 1 signal[z > entry_z] = -1 return signal

Faktor-baseret portefølje — byg på forskning.

Akademisk veldokumenterede faktorer der historisk gav premium:

Value

Lav P/E, P/B. Køb billige, undgå dyre.

Quality

Høj ROE, lav gæld, stabile margins. Buffett-stilen.

Momentum

Seneste 12 måneders relative styrke.

Low volatility

Anomali: lavvolume aktier giver bedre risk-adjusted afkast.

Size

Smaa-cap premie historisk — men reduceret efter 1980.

Multi-factor screen
def factor_score(stock_data): """Komb. score over flere faktorer.""" # Hver faktor: rank stocks fra 0-1 value_rank = stock_data['pe_ratio'].rank(pct=True, ascending=True) quality_rank = stock_data['roe'].rank(pct=True, ascending=False) momentum_rank = stock_data['returns_12m'].rank(pct=True, ascending=False) lowvol_rank = stock_data['volatility_3m'].rank(pct=True, ascending=True) # Lige vægt mellem faktorer composite = (value_rank + quality_rank + momentum_rank + lowvol_rank) / 4 return composite # Køb top decile (10%), shorte bottom (long-short) top_decile = stocks[composite_score > 0.9] bottom_decile = stocks[composite_score < 0.1]

Røde flag i alpha-strategier — de fleste største-end-Sharpe-3 er falske.

  • For høj frekvens. Hvis strategien handler dagligt med Sharpe 5.0, har du sandsynligvis ikke modelleret costs realistisk.
  • Lille univers. Strategi virker på 5 aktier. Det er ikke statistik, det er anekdoter.
  • Kort backtest. 2-3 års data er ikke nok. Sigt mod 10+ år over flere markedscyklusser.
  • Sharpe falder dramatisk uden enkelt periode. Hvis 2020-COVID-rally driver hele afkastet, ikke alpha.
  • Lookahead-bias. Strategi bruger data den ikke ville have haft på beslutningsmomentet.
Realistisk forventningProfessionelle hedgefonds opnår Sharpe 0.8-1.5 over lange perioder. Din strategi i backtest med Sharpe 3.5 er sandsynligvis overfittet eller har en bug. Vær skeptisk.

Diversifikation — tre principper.

  • Adskilte alpha-kilder. Momentum + mean reversion + value handler i forskellige regimer.
  • Tids-diversifikation. Strategier med forskellige holding-perioder (dag, uge, måned, kvartal).
  • Geografisk & asset-class. US-aktier + EU-aktier + bonds + commodities adskiller risk.

Hvor AI fejler

  • En enkelt strategi all-in. Når den fejler, fejler alt. Diversificér.
  • Stoler på backtest med Sharpe 4+. Er næsten altid overfittet eller bug.
  • Ignorerer hvad markedet betaler for. Risk premia eksisterer kun fordi nogen betaler dem. Forstå hvorfor.
  • Skifter strategi efter et dårligt halv-år. Selv gode strategier har 1-2 års underperformance-perioder.
  • Realistic alpha: Sharpe 0.8-1.5 lang sigt. Højere er mistanke.
  • Diversificér mellem alpha-kilder, ikke kun aktier.
  • Vær mest skeptisk overfor strategier der altid virker.
  • Lang holding-periode (måneder) skalerer bedre end intraday.

Risk & performance

Risiko er ikke volatilitet.

Klassisk teori siger risiko = standardafvigelse af afkast. Det er en bekvem matematisk antagelse, ikke en korrekt definition. Reel risiko er sandsynligheden for at miste mere end du kan tåle — permanent kapital-tab, ikke daglige bevægelser.

De tre risk-niveauer du må måle.

Position-niveau

Hvor meget kan denne enkelte position tabe? Concentration risk.

Strategi-niveau

Hvor meget kan denne strategi tabe (max drawdown)? Strategy-specific risk.

Portefølje-niveau

Hvor meget kan hele porteføljen tabe? Aggregate tail risk.

Value at Risk (VaR) — klassisk men begrænset.

"Med 95% sandsynlighed taber jeg ikke mere end X i en dag." Nyttigt som heuristik, farligt som eneste mål:

VaR-beregning
def historical_var(returns, confidence=0.95): """Historisk VaR: empirisk percentile.""" return np.percentile(returns, (1-confidence)*100) def parametric_var(returns, confidence=0.95): """Parametrisk VaR antagende normalt fordelt.""" from scipy import stats z = stats.norm.ppf(1-confidence) return returns.mean() + z * returns.std() def cvar(returns, confidence=0.95): """Conditional VaR / Expected Shortfall. Forventet tab GIVET at vi rammer worst (1-conf)%.""" var = historical_var(returns, confidence) return returns[returns <= var].mean() # Eksempel daily_returns = portfolio_returns print(f'95% daglig VaR: {historical_var(daily_returns, 0.95):.4f}') print(f'95% daglig CVaR: {cvar(daily_returns, 0.95):.4f}')

VaR's fatale begrænsning.

VaR siger intet om hvor slemt det kan blive i de værste 5%. Det er forskellen mellem "tabt 1.000 kr" og "tabt 100.000 kr" — begge ligger over 95% VaR-tærsklen. Brug derfor altid CVaR (Expected Shortfall) ved siden af.

Tail risk — det der dræber dig.

Markedsafkast er ikke normalfordelt. De har "fat tails" — ekstreme events sker langt oftere end normalfordelingen tilsiger. 1987-crashet, 2008, 2020 — alle var teoretisk umulige hver 10.000+ år:

Stress test

Hvad ville porteføljen tabe i 2008-replay? 1987? COVID?

Skewness

Asymmetri af afkast. Negativ skew = hyppige små gevinster, sjældne store tab.

Kurtosis

Tail-heaviness. Højere kurtosis = flere ekstreme events.

Tail hedging

Køb billige out-of-the-money puts som forsikring mod store fald.

Position sizing — vigtigere end strategi.

De fleste fokuserer på hvad de skal købe. De professionelle fokuserer på hvor meget. Position sizing er hvor 80% af risk-management ligger:

Equal weight

Hver position = 1/N af kapital. Simpelt, robust.

Volatility weight

Større position i lavvolume, mindre i højvolume. Risikolikvider.

Kelly criterion

Optimal vækst hvis du kender win-rate og r/r. Tend ofte for aggressiv.

Risk parity

Hver position bidrager lige til portefølje-volatilitet.

Volatility-weighted positions
def vol_target_sizing(returns, target_vol=0.10, lookback=63): """Position-størrelse så hver position bidrager samme vol.""" realized_vol = returns.rolling(lookback).std() * np.sqrt(252) # Skalerér position op når vol lav, ned når vol høj leverage = target_vol / realized_vol.iloc[-1] return min(leverage, 2.0) # max 2x leverage

Korrelations-risk — diversifikation forsvinder når man har brug for den.

I rolige markeder er korrelationer 0.3. I kriser stiger de til 0.8-0.9. Alle dine "uafhængige" positioner falder sammen. Diversifikation virker mindst når du har mest brug for den.

RealismeAntag ikke diversifikation virker i kriser. Hav direkte tail hedges (puts, gold, cash) der ikke korrelerer med risk assets.

Performance-attribution — forstå dine afkast.

Et 15% årligt afkast er ikke nok information. Du må forstå HVOR afkastet kom fra:

  • Markedets bidrag (beta). Hvor meget kom bare fra at være i markedet?
  • Faktor-bidrag. Value, momentum, size — kommer afkastet fra kendte faktorer?
  • Ægte alpha (residual). Det der ikke kan forklares af ovenstående.
  • Heldfaktor (luck). Sjældent kan du adskille alpha fra held på mindre end 5 års data.
Fama-French regression

Hvor AI fejler

  • Måler kun volatilitet. Misser tail risk og concentration risk.
  • Antager normalfordeling. Markedsafkast har fat tails. VaR underestimerer.
  • Ignorerer korrelation-stigning i kriser. "Diversificeret" portefølje falder samlet.
  • Ingen attribution-analyse. Ved ikke om afkast var alpha eller bare beta.
  • CVaR (Expected Shortfall) slaar VaR — fortæller dig hvor slemt det kan blive.
  • Position sizing er vigtigere end position-valg.
  • Antag korrelationer stiger til 0.8+ i kriser. Hav direkte tail hedges.
  • Attribution-analyse afslører om dit afkast er alpha eller held.

Live & paper

Backtest til live er som tegning til byggeri.

Det meste af forskellen mellem en flot backtest og et fungerende live-system: 100 små ting der ikke betyder noget i simulering men er kritiske i produktion. Latency, ordre-rækkefølge, partial fills, dataforsinkelser, broker-fejl. Den disciplinerede vej er paper trading i 3-6 måneder før live.

De fire deployment-stadier.

1. Backtest

Historisk data, alt antaget. Hurtig iteration, store antagelser.

2. Forward test

Kør strategi på ny data i realtid, men uden faktiske handler. Validerer signal-generering.

3. Paper trading

Send "fake orders" til broker. Få ægte slippage, partial fills. Ingen rigtige penge.

4. Live (small)

Lille kapital. Find produktionsfejl med begrænset skade. 2-3 måneder.

5. Live (full)

Op til ønsket allokering når alt er valideret.

Bind dig ikke til én broker.

Forskellige brokere har vidt forskellige API-kvaliteter:

Interactive Brokers

Industri-standard for kvant. Lavt commission, omfattende API. Stejlt læringskurve.

Alpaca

US-only. Gratis kommission. Nem REST-API. God til at lære.

Saxo Bank

Dansk. Solid produktportefølje. OpenAPI tilgængelig.

Tradier

Options-fokuseret. Lav commission. US-only.

Live trading-arkitektur.

Minimal live trader
import asyncio from datetime import datetime import logging class LiveTrader: def __init__(self, strategy, broker, max_position_value=10000): self.strategy = strategy self.broker = broker self.max_position_value = max_position_value self.positions = {} self.logger = logging.getLogger(__name__) self.kill_switch = False async def run(self): """Main event loop.""" while not self.kill_switch: try: # 1. Tjek om markedet er åbent if not await self.broker.is_market_open(): await asyncio.sleep(60) continue # 2. Hent seneste data data = await self.broker.get_latest_data() # 3. Generer signaler signals = self.strategy.generate_signals(data) # 4. Risk checks FØR execution if not self.passes_risk_checks(signals): self.logger.warning('Signals failed risk checks') continue # 5. Execute trades for symbol, target_size in signals.items(): await self.execute_trade(symbol, target_size) # 6. Log state await self.log_portfolio_state() # 7. Sleep til næste iteration await asyncio.sleep(self.strategy.signal_interval) except Exception as e: self.logger.error(f'Loop error: {e}', exc_info=True) await self.send_alert(f'Trader error: {e}') # KILL SWITCH på alvorlige fejl if isinstance(e, (BrokerError, DataIntegrityError)): self.kill_switch = True def passes_risk_checks(self, signals): # Max position value for symbol, target in signals.items(): if abs(target * self.broker.get_price(symbol)) > self.max_position_value: return False # Daily loss limit daily_pnl = self.compute_daily_pnl() if daily_pnl < -self.max_daily_loss: self.logger.error(f'Daily loss limit reached: {daily_pnl}') self.kill_switch = True return False # Max gross exposure gross = sum(abs(p * self.broker.get_price(s)) for s, p in self.positions.items()) if gross > self.max_gross_exposure: return False return True async def execute_trade(self, symbol, target_size): current = self.positions.get(symbol, 0) delta = target_size - current if abs(delta) < 1: return # Brug limit orders med max slippage current_price = await self.broker.get_price(symbol) limit_price = current_price * (1 + 0.001 if delta > 0 else 1 - 0.001) order = await self.broker.place_order( symbol=symbol, quantity=delta, order_type='LIMIT', limit_price=limit_price, time_in_force='IOC', # Immediate-or-cancel ) if order.status == 'FILLED': self.positions[symbol] = target_size self.logger.info(f'Filled: {symbol} {delta} @ {order.avg_price}') else: self.logger.warning(f'Partial/unfilled: {symbol} {order.filled_qty}/{delta}')

Kritiske safety mechanisms.

Daily loss limit

Hvis tab i dag > 2% af kapital, sluk handler resten af dagen.

Position size cap

Aldrig over X% i én position, uanset hvad signalet siger.

Gross exposure cap

Hvis leveraged: max 1.5x bruttoeksponering.

Kill switch

En knap der lukker alle positioner og slukker.

Heartbeat-monitor

Hvis dit system ikke har "pinget" i 5 min, alert + auto-flatten.

Order-rate limit

Maks N ordrer pr. minut. Forhindrer at en bug fyrer 10.000 ordrer af.

Det vigtigste princip: fail safe.

Når noget går galt — og det gør det — skal systemet defaulte til "ingen positioner" frem for "fortsæt handler". Eksempler:

  • Datafeed går ned → lukker positioner, stopper
  • Broker API utilgaengelig → forsøger reconnect, men ingen nye ordrer
  • Strategy returnerer NaN eller infinity → log fejl, skip iteration
  • Disk fyldt op → alert, men fortsæt én iteration før stop

Monitoring & alerts.

  • PnL real-time. Du må vide hvor du står på sekunder.
  • Position drift. Faktiske positioner vs forventede — advarsel ved divergens.
  • Latency-tracking. Hvor lang tid tager signaler → ordrer → execution?
  • Slippage-attribution. Hvor meget tabte vi på udførelses-priser?
  • Alerts via SMS/Slack/PagerDuty. Når limits overskrides eller fejl opstår.

Reconciliation — matching internal vs broker.

Mindst dagligt: sammenlign hvad dit system tror du ejer mod hvad brokeren rapporterer du ejer. Divergens er ofte tegn på missede fills, hidden commissions, eller fejl i din egen tracking. Tag det alvorligt.

Hard ruleHvis broker-state og dit interne state divergerer i mere end 5 min: stop handler. Investigér før restart.

Hvor AI fejler

  • Skip paper trading. Live med 100% kapital direkte fra backtest. Garanti for at finde bugs i prod.
  • Ingen kill switch. Bug fyrer 1000 ordrer af. Du har ikke en knap der stopper det.
  • Ingen reconciliation. Phantom positions. Du tror du har solgt. Du har ikke.
  • Trader 24/7. Søvn er nødvendigt. Hav monitoring/alerts der vækker dig kun ved kritiske ting.
  • Paper trading mindst 3 måneder før live. Ingen genveje.
  • Fail safe — default til "ingen positioner" ved fejl.
  • Reconcile dagligt mod broker. Divergens er bug.
  • Kill switch + daily loss limit er ikke valgfri.
